# Критичний аналіз експериментів та рекомендації щодо оновлення оцінки якості ArUco-маркерів

У цьому документі проведено детальний аналіз результатів експериментального тестування маркерів (із файлу `utils/marker_quality_test/log.txt`) та надано рекомендації щодо модифікації алгоритму оцінки.

---

## 1. Критичний аналіз результатів експериментів

Експериментальні випробування 72 маркерів, результати яких зафіксовано у `utils/marker_quality_test/log.txt`, загалом підтверджують працездатність нашої ML-моделі ранжування. Маркери, які отримали високі оцінки $Q$, переважно демонструють стабільність на рівні $99\% - 100\%$, тоді як низькооцінені маркери падають до $67\% - 75\%$.

Проте було виявлено **критичні аномалії**:
*   **ID 599**: Прогнозована оцінка $Q = 0.9141$ (висока якість), але реальний показник розпізнавання становить лише **83.80%**.
*   **ID 598**: Прогнозована оцінка $Q = 0.8068$ (прийнятна якість), але реальний показник становить **85.18%**.

### 1.1 Пошук прихованого фактора: Аналіз зв'язності чорного кольору
Детальний аналіз геометричних властивостей маркерів-аномалій та їх порівняння з елітною групою дозволив виявити новий критичний фактор — **кількість зв'язних компонент чорного кольору всередині матриці (`BlackComponents`)**.

Статистичний аналіз вибірки з `log.txt` показав:
1.  **Кореляція**: Кількість чорних компонент має **найсильніший негативний зв'язок** із реальним показником Detection Rate серед усіх метрик ($r = -0.567$).
2.  **Аналіз лідерів**: Усі маркери з Топ-10 експерименту (ID 691, 698, 575, 703, 696, 705, 689 тощо) мають **рівно 1 чорну зв'язну компоненту** (`BlackComponents = 1`).
3.  **Аналіз аутсайдерів**: Майже всі маркери з найгіршими показниками розпізнавання (включаючи аномалії ID 598 та ID 599) мають **2 або більше чорні зв'язні компоненти** (`BlackComponents >= 2`).

### 1.2 Фізичне пояснення аномалії
ArUco-маркери мають зовнішню чорну рамку шириною в 1 клітинку. Внутрішня чорна геометрія матриці 4x4 має зливатися із цією рамкою, створюючи єдиний суцільний чорний контур. 
Якщо чорні клітинки всередині матриці розділені білими клітинками на дві або більше ізольованих областей (`BlackComponents >= 2`), це означає, що:
- Білі області прорізають чорну структуру наскрізь або ізолюють частину чорних клітинок.
- При оптичному розмитті камери або нахилі такі розділені чорні блоки зливаються з білим фоном, порушуючи топологічну цілісність рамки і збиваючи порогові алгоритми сегментації контуру.

## 2. Рекомендації щодо оновлення програми визначення ID найкращих маркерів

Для усунення виявлених помилок прогнозування необхідно оновити програмний пайплайн у каталозі `utils/evaluation/`.

### 2.1 Модифікація модуля ознак (`features.py`)
Необхідно додати обчислення нових ознак:
1.  **`BlackComponents`**: Кількість зв'язних компонент чорного кольору (значення 0) у матриці 4x4 за допомогою пошуку в ширину (BFS).
2.  **`TotalComponents`**: Сума чорних та білих компонент (`BlackComponents + WhiteComponents`).

### 2.2 Впровадження жорсткого фільтра (Hard Constraint)
Враховуючи, що наявність більше ніж однієї чорної компоненти є деструктивною, у модуль оцінювання `evaluate.py` необхідно інтегрувати жорстке правило:
*   Якщо маркер має `BlackComponents > 1`, його оцінка якості $Q$ автоматично штрафується або маркер повністю виключається зі списку кандидатів (наприклад, $Q = 0.0$), незалежно від інших ідеальних параметрів.

### 2.3 Перенавчання моделі (`train.py`)
Необхідно додати `BlackComponents` та `TotalComponents` до переліку ознак навчання моделі та запустити повторне навчання на розширеному наборі з 294 маркерів. Завдяки дуже високій кореляції цієї ознаки з реальними даними, модель логістичної регресії отримає набагато точніші та фізично адекватні вагові коефіцієнти, що повністю виключить вибір маркерів типу ID 598 та ID 599 як "хороших".
