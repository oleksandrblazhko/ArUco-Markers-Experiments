# Опис алгоритму роботи програми Plane Quality Assessment

Ця програма призначена для оцінювання просторової якості розпізнавання ArUco-маркерів у межах поля зору камери. Вона дозволяє побудувати карту стабільності детекції та визначити області зниженої точності, що є критично важливим для завдань комп'ютерного зору.

## Аргументи командного рядка

Програма підтримує наступні параметри конфігурації, які можна вказати при запуску:

*   `--cam <ID>`: ID пристрою камери (за замовчуванням: `0`).
*   `--width <WIDTH>`: Ширина кадру камери (за замовчуванням: `640`).
*   `--height <HEIGHT>`: Висота кадру камери (за замовчуванням: `480`).
*   `--flip`: Прапорець, який, якщо встановлений, горизонтально віддзеркалює зображення з камери.
*   `--marker-id <ID>`: ID ArUco-маркера, що використовується в сітці (за замовчуванням: `691`).
*   `--marker-size <SIZE>`: Розмір сторони ArUco-маркера в міліметрах (за замовчуванням: `8.0`).
*   `--marker-separation <SEP>`: Розмір білої смужки (проміжку) між маркерами в міліметрах (за замовчуванням: `1.0`).
*   `--grid-rows <ROWS>`: Кількість рядків у матриці маркерів (за замовчуванням: `19`).
*   `--grid-cols <COLS>`: Кількість стовпців у матриці маркерів (за замовчуванням: `27`).

## Алгоритм роботи програми

Алгоритм роботи програми складається з двох основних фаз:

### Фаза 1: Пошук та стабілізація гомографії (Plane Homography Discovery and Stabilization)

Оскільки всі маркери на тестовому аркуші мають однаковий ID, програма не може використовувати унікальні калібрувальні маркери для визначення положення площини аркуша. Натомість, вона автоматично визначає перспективне перетворення (гомографію) у реальному часі.

1.  **Детекція початкового кластера маркерів:** Програма аналізує вхідні відеокадри, виявляючи всі ArUco-маркери із заданим `--marker-id`.
2.  **Ідентифікація кутових маркерів:** Коли виявляється достатня кількість маркерів (більше 20), програма ідентифікує чотири маркери, що знаходяться на краях виявленого кластера. Вони використовуються як початкові точки для встановлення положення аркуша.
3.  **Розрахунок початкової гомографії:** На основі піксельних координат цих чотирьох кутових маркерів та їхніх відомих ідеальних світових координат (визначених параметрами `--marker-size`, `--marker-separation`, `--grid-rows`, `--grid-cols`) розраховується приблизна матриця гомографії.
4.  **Уточнення гомографії (RANSAC):** За допомогою цієї початкової гомографії програма прогнозує піксельні позиції всіх 513 маркерів гіпотетичної сітки. Потім вона зіставляє ці прогнозовані позиції з усіма фактично розпізнаними маркерами на кадрі. Використовуючи метод RANSAC, розраховується уточнена та стабільна матриця гомографії, яка найкраще відповідає всім відповідним точкам.
5.  **Блокування гомографії:** Після успішного знаходження стабільної гомографії програма переходить до наступної фази, фіксуючи цю гомографію.

### Фаза 2: Збір даних та аналіз якості (Data Collection and Quality Analysis)

На цій фазі програма використовує вже стабілізовану матрицю гомографії для систематичного збору даних про якість розпізнавання по всій поверхні тестового аркуша.

1.  **Генерація гіпотетичної сітки:** На початку цієї фази (або один раз після стабілізації гомографії) програма створює повну модель ідеального розташування всіх 513 маркерів (19x27) у міліметрових координатах.
2.  **Проекція сітки:** На кожному новому відеокадрі ця гіпотетична сітка проектується на площину зображення камери за допомогою стабілізованої гомографії. Таким чином, для кожної з 513 позицій маркерів ми знаємо їхнє очікуване місцезнаходження в пікселях.
3.  **Зіставлення та логування:** Для кожної прогнозованої позиції маркера програма намагається знайти відповідний фактично розпізнаний маркер.
    *   **Якщо маркер розпізнано:** Збираються метрики якості (різкість, розмір маркера в пікселях) та статус "розпізнано". Ці дані зберігаються як `PlaneSample`, використовуючи *ідеальні* міліметрові координати з гіпотетичної сітки для стабільного групування.
    *   **Якщо маркер НЕ розпізнано:** Записується `PlaneSample` зі статусом "не розпізнано" та нульовими значеннями метрик. Це гарантує, що кожна позиція на сітці буде врахована у фінальному аналізі, навіть якщо маркер там відсутній або невидимий.
4.  **Візуалізація в реальному часі:** На відеокадрі відображаються маленькі кружечки: зелені для розпізнаних маркерів і червоні для нерозпізнаних, показуючи візуальну карту якості в реальному часі.
5.  **Збір кадрів:** Усі зібрані `PlaneSample` групуються в об'єкти `PlaneFrame`, які, у свою чергу, збираються в `PlaneQualityCollector` для подальшого аналізу.

## Вихідні файли

Після завершення збору даних (натискання `ESC`) програма генерує наступні файли з результатами:

*   `plane_quality.csv`: Зведені статистичні дані в форматі CSV.
*   `plane_quality.json`: Зведені статистичні дані в форматі JSON.
*   `plane_quality_grid.md`: Markdown-файл, що містить табличне представлення сітки 19x27, де для кожної позиції вказано відсоток розпізнавання.

## Приклад використання

Для запуску програми з використанням камери з ID `0`, розміром кадру `1280x720`, маркерами 10 мм, проміжком 2 мм, сіткою 20x30 та дзеркальним відображенням:

```bash
python main.py --cam 0 --width 1280 --height 720 --marker-size 10.0 --marker-separation 2.0 --grid-rows 20 --grid-cols 30 --flip
```
