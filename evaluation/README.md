### 1. Навчання моделі

  Перед початком оцінювання маркерів необхідно навчити модель машинного навчання на основі наявних експериментальних даних:

    python train.py

  Що робить цей скрипт:
1) Автоматично завантажує експериментальні дані з файлу marker_detection_rate.json.
2) Обчислює 10 геометричних ознак для всіх маркерів.
3) Порівнює моделі логістичної регресії та випадкового лісу за допомогою крос-валідації:
- Крос-валідація дозволяє перевірити, як моделі будуть працювати на нових маркерах, що не брали участі у навчанні.
- ROC-AUC оцінює здатність моделі розділяти "хороші" (detection rate ≥95%) та "погані" маркерів (detection rate <95%).
4) Зберігає найкращу модель у файл  model.pkl .

  ### 2. Оцінка та ранжування маркерів

  Після успішного навчання моделі використовуйте скрипт  evaluate.py  для розрахунку оцінки якості Q:

  • Оцінити діапазон маркерів (наприклад, від 0 до 20):
    python evaluate.py 0-20

  • Оцінити конкретні ID маркерів:
    python evaluate.py 13,17,45,102

  • Використати інший словник (наприклад, 4X4_1000) та обмежити вивід топ-5 найкращими кандидатами:
    python evaluate.py 0-100 --dict 4X4_1000 --limit 5


  Що робить цей скрипт:

  • Генерує бітові матриці маркерів та обчислює їх ознаки.
  • Прогнозує ймовірність Q ∈ [0,1] того, що маркер матиме стійкість розпізнавання ≥95%.
  • Виводить відсортовану за спаданням якості таблицю та список упорядкованих ID через кому.
